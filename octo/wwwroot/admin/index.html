How it works

The shim is its own container running yt-dlp behind a tiny Flask wrapper. Process isolation keeps yt-dlp's quirks — and YouTube's frequent extractor breakage — from affecting Octo. Concurrent yt-dlp slots and cache size are configured via env vars on the yt-dlp-shim service in docker-compose.yml.

Raw config

Edit the settings file directly. Same JSON the form-by-form UI writes to — useful for power-users who'd rather hand-write a section than click through tabs, or for diffing/copying config across hosts.

File path

The file lives at /app/config/settings.json inside the container. The host bind-mount is at ./octo-config/settings.json next to your docker-compose.yml — you can vim/nano it there too. Changes hot-reload within ~500ms; settings flagged restart-required still need a restart.

settings.json

Top-level keys: Subsonic, Library, Soulseek, YouTube, LastFm. Anything you set here overrides the corresponding env var until removed.

Config sources

About

What this is and where things live.

Octo is a Subsonic API proxy that augments your existing Navidrome library with discovery and on-demand downloading. Search includes external recommendations alongside your local songs; "Radio" on any song fans out via Last.fm and previews via YouTube; star a song you don't own and Octo grabs it via Soulseek.

Settings file

Anything you change in this UI is persisted to /app/config/settings.json. That file overrides env vars on next read. Editing it by hand also works — changes hot-reload.

Hot-reload vs restart

Most settings (storage mode, folder structure, filters) take effect immediately. Settings flagged restart required bake into HTTP clients at startup; use the Restart button in the sidebar to apply them.
